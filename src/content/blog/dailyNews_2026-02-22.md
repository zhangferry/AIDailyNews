### [AI现场生成软件将取代应用商店？Karpathy对未来软件形态的预判](https://www.infoq.cn/article/4LPuZzDH4CXh5G5h5N4G)

来源：InfoQ 推荐

发布时间：2026-02-22 20:37:45

背景：Andrej Karpathy 分享了其利用 AI Agent 仅用一小时便生成个性化健身追踪仪表盘的实验，并对比了传统应用商店模式的局限性。核心观点：随着模型能力提升，软件正从“标准化产品”演变为“按需生成的短暂工具”。当前制约生成速度的主要瓶颈在于基础设施缺乏 AI 原生 API，而非模型理解力。未来自然语言将成为编程接口，通过 Agent 调用标准化技能库，软件有望在 1 分钟内生成。结论：应用商店不会彻底消失，但角色将弱化为提供经过验证的基础能力层，真正的竞争力将转向 AI 系统的部署速度与基础设施的完善度。软件将变成一种“即用即弃”的服务。

### [LocalStack 取消开源社区版，转向单一镜像注册模式](https://www.infoq.cn/article/Ek2SwBQSUmCAbg9V7WWD)

来源：InfoQ 推荐

发布时间：2026-02-22 08:00:00

背景：云服务模拟器 LocalStack 宣布调整交付模式，取消了广受欢迎的免费开源社区版，计划将其与专业版合并为一个单一镜像。核心观点：官方称此举是由于维护高保真 AWS 仿真的安全与运营成本日益增加，需引入基于账户的注册模式以实现可持续发展。新模式将提供有限的免费层和付费 CLI v2，停止对旧社区版的更新。结论：目前依赖 Docker Hub 自动拉取最新社区版镜像的项目需紧急更新工作流。这一变动引发了社区对开源承诺变质的强烈不满，开发者正寻找如 Moto 等替代方案。

### [ClaudeBar：macOS AI 编码助手配额监控工具](https://github.com/tddworks/ClaudeBar)

来源：iOS Development News - Telegram Channel

发布时间：2026-02-22 10:17:33

开发者常需同时使用多种 AI 编码助手（如 Claude、Copilot 等），分散的配额管理难以实时掌握用量，可能影响开发效率。ClaudeBar 是一款 macOS 菜单栏应用，旨在集中监控多种 AI 编码助手的使用配额。它支持 Claude、Codex、Gemini、Kimi 等主流服务，通过集成底层 CLI 工具或 API 探针获取数据。核心功能包括实时显示会话、周级及模型维度的用量百分比，支持多主题切换、系统通知及自定义阈值预警。项目采用 SwiftUI 和 TDD 架构，设计分层清晰，支持通过 Protocol 注入依赖进行扩展。该工具为多工具并行开发场景提供了统一的配额管理视图，有效防止 API 额度耗尽导致的服务中断，提升了开发资源管理的精细化水平。

### [DeepLink Kit：iOS 深度链接路由处理库](https://github.com/button/DeepLinkKit)

来源：iOS Development News - Telegram Channel

发布时间：2026-02-22 10:12:14

在 iOS 开发中，深度链接的处理涉及 URL Scheme 注册、参数解析、路由分发及页面跳转，传统手动实现方式代码冗余且易出错。DeepLink Kit 是一个专注于路由匹配和 Block 调用的 iOS 深度链接处理库。它允许开发者通过简单的语法注册路由规则，支持复杂的正则表达式匹配和命名参数提取，从而自动处理 URL 解析逻辑。库内建了对 Universal Links 和 AppLinks 的支持，并能处理应用间的复杂对象传递。通过将路由逻辑集中管理，解耦了业务代码与链接处理逻辑。该库显著降低了 iOS 应用集成深度链接的复杂度，提供了一套标准化的解决方案来处理唤醒和导航逻辑，增强了代码的可维护性。
